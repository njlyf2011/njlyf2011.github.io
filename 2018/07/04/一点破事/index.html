    一点破事

    2018-07-04
    个人

    今天在某 Linux 群吹水，发现有人发了一张图。图中的那个人说他很厉害，能看到完整的 NT10 的源码。

    我于是找他要了群号，进了群，发现图里的那位还在吹嘘，说自己是“微软系统架构师”，后来我说，能发一下 NT5 源码么，截图就行。他发了一张图，说是 NT 5.x 的源码。
    图里的源码提到了 shell32 之类的内容，如果不仔细看的话还真以为是 Windows 的，不过，他忘记遮盖了 rostests 字样，也就是说，通过遮盖可以看出这个来自 ReactOS 的测试套件。（注：ReactOS 是一套开源的类 Windows 操作系统，来自对 NT 的反向工程）

    不用多说了，这个肯定是假的，Windows 里面不可能出现 ReactOS 测试套件的内容。于是我说明了我的真实情况，也告诉了他我参与过 ReactOS 开发的事并戳穿了他的谎言。

    他恼羞成怒，并嘲讽般的说了好几遍 “好厉害哦”，并继续强调自己是“微软工程师”，能看到所有源码。后来我说，就算是微软的人，因为保密措施，也不可能看到所有源码，他开始说我“不懂微软”，虽然口气还是那么狂妄，但已经看出，他的辩解变得越来越苍白无力。后来，他改了口，说自己“没有源码只有 0 day”。

    自从我混 Linux 圈到现在，我见过不少这种“技术大佬”，像他这种，吹嘘自己有源码的，也见过。某次我见过的一个“技术大佬”，自称获得过 Windows 10 源码并将其移植到了 Linux 上使之成为子系统，又说自己编写了一套操作系统内核，而且是在短短几个月时间内说的，一个人（自称一个人），在短短几个月之内完成了两项这么重的工作，怎么可能，很明显就是假的。

    自己什么都不是却给自己制造光环，这种行为真的虚伪至极。而且，既然他已经对我们谎称他的这些经历，那么，为了让我们继续相信，那么他肯定需要一而再再而三圆谎，我一直就不信，谎言可以永远圆下去，迟早有编不下去真相大白的一天。
